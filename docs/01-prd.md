# Tab Pet PRD

## 1. 제품 개요

Tab Pet은 브라우저 탭의 `document.title`과 favicon을 살아있는 펫 UI처럼 활용하는 단일 페이지 웹서비스이다. 사용자는 업무 탭처럼 보이는 페이지를 열어두고, 실제로는 탭 안에서 고양이 또는 강아지를 돌본다.

본문 UI는 설정과 액션을 수행하는 보조 인터페이스이며, 핵심 경험은 탭 제목과 favicon의 변화다.

## 2. 문제 정의

대부분의 웹 기반 펫/타이머/힐링 서비스는 사용자가 페이지 본문을 직접 보고 있을 때만 의미가 있다. 하지만 실제 업무 환경에서는 사용자가 여러 탭을 열어두고 이동하기 때문에, 탭 자체가 상태를 전달하면 더 가볍고 지속적인 경험을 만들 수 있다.

Tab Pet은 브라우저 탭을 작은 생활 공간처럼 사용한다. 펫의 배고픔, 졸림, 지루함, 기분 상태가 탭 제목과 favicon에 반영되어 사용자가 페이지를 열지 않아도 상태를 감지할 수 있게 한다.

## 3. 목표

- 브라우저 탭 제목과 favicon을 핵심 UI로 사용하는 펫 MVP를 만든다.
- 고양이/강아지 선택, 스탯, 액션, 저장/복원, 오프라인 decay를 구현한다.
- 업무 탭처럼 보이는 위장 타이틀 프리셋을 제공한다.
- 스킨/테마, 광고 보상형, 프리미엄 UI, 이모지 복사 패널은 실제 과금 없이 목업으로 제공한다.
- 웹서비스로 먼저 검증하고, 크롬 확장은 후속 검토 대상으로 남긴다.

## 4. 비목표

이번 MVP에서는 다음을 구현하지 않는다.

- 로그인
- 서버 DB
- 멀티 펫
- 소셜 기능
- 푸시 알림
- 실제 광고 SDK
- 실제 결제
- 크롬 확장

## 5. 핵심 컨셉

### 5.1 위장형 탭 펫

사용자는 Tab Pet을 업무용 탭처럼 열어둘 수 있다. 예를 들어 탭 제목은 `Project Dashboard`, `Quarterly Report`, `Inbox`처럼 보이지만, 뒤에 붙는 기호나 favicon 변화로 펫의 상태를 알아볼 수 있다.

### 5.2 본문보다 탭 반응 우선

본문 UI는 사용자가 펫에게 액션을 수행하거나 설정을 바꾸는 화면이다. 하지만 제품 차별점은 본문 UI가 아니라 브라우저 탭 자체다. 따라서 기획, 설계, QA 기준 모두 title/favicon 반응을 우선한다.

### 5.3 가벼운 로컬 경험

MVP는 로그인 없이 `localStorage`만 사용한다. 사용자는 같은 브라우저에서만 상태가 유지된다. 이는 MVP 범위를 줄이고, 탭 반응이라는 핵심 가설을 빠르게 검증하기 위한 선택이다.

## 6. 대상 사용자

### 6.1 1차 사용자

- 업무 중 브라우저 탭을 오래 켜두는 사용자
- 귀여운 생산성 보조 도구를 좋아하는 사용자
- 회사나 학교 환경에서 노골적인 게임 화면보다 조용한 위장형 UI를 선호하는 사용자

### 6.2 사용자 니즈

- 눈에 띄지 않게 작은 재미를 유지하고 싶다.
- 탭만 봐도 펫 상태를 알 수 있으면 좋겠다.
- 복잡한 가입 없이 바로 시작하고 싶다.
- 펫을 돌보는 행동이 너무 무겁거나 방해되지 않았으면 한다.

## 7. MVP 범위

### 7.1 펫 선택

사용자는 최초 진입 시 고양이 또는 강아지를 선택한다.

선택 항목:

- Cat
- Dog

선택 이후 기본 스탯으로 펫 상태가 생성된다.

### 7.2 스탯

펫은 세 가지 스탯을 가진다.

| 스탯 | 범위 | 의미 |
| --- | --- | --- |
| `fullness` | 0-100 | 배부름 |
| `mood` | 0-100 | 기분 |
| `energy` | 0-100 | 에너지 |

스탯은 저장 대상이다. 상태값은 저장하지 않고 스탯에서 계산한다.

### 7.3 상태

MVP 상태는 다음 여섯 가지다.

| 상태 | 의미 |
| --- | --- |
| `happy` | 기본적으로 안정적인 상태 |
| `hungry` | 배고픈 상태 |
| `sleepy` | 에너지가 낮은 상태 |
| `bored` | 기분이 낮아진 상태 |
| `sad` | 기분이 매우 낮은 상태 |
| `excited` | 컨디션이 매우 좋은 상태 |

### 7.4 액션

사용자는 세 가지 액션을 수행할 수 있다.

| 액션 | 효과 |
| --- | --- |
| `feed` | 배부름 증가, 기분 소폭 증가, 에너지 소폭 감소 |
| `play` | 기분 증가, 배부름 감소, 에너지 감소 |
| `sleep` | 에너지 증가, 배부름 감소, 기분 소폭 증가 |

모든 액션은 스탯을 0-100 범위로 clamp한다.

### 7.5 위장 타이틀 프리셋

사용자는 탭 제목의 기본 문구를 고를 수 있다.

초기 프리셋:

- Project Dashboard
- Quarterly Report
- Inbox
- Analytics
- Untitled Document
- Meeting Notes

상태별 시그널은 프리셋 뒤에 작은 기호로 붙인다. MVP에서는 사용자가 보기에 너무 노골적인 펫 문구를 title에 직접 노출하지 않는다.

예시:

| 상태 | title 예시 |
| --- | --- |
| `happy` | `Project Dashboard` |
| `hungry` | `Project Dashboard *` |
| `sleepy` | `Project Dashboard ...` |
| `bored` | `Project Dashboard -` |
| `sad` | `Project Dashboard !` |
| `excited` | `Project Dashboard +` |

### 7.6 favicon 반응

favicon은 펫 종과 상태에 따라 변경된다.

MVP 구현 방식:

- 상태별 색상과 표정을 가진 SVG favicon을 런타임 생성한다.
- 기존 favicon link 태그를 찾아 교체하거나 없으면 새로 생성한다.
- 브라우저 환경에서만 실행한다.

### 7.7 저장/복원

`localStorage`에 현재 펫 상태를 저장한다.

저장 대상:

- 펫 종
- 스탯
- 선택한 위장 타이틀 ID
- 선택한 테마 ID
- 마지막 업데이트 시각
- 앱 데이터 버전

복원 시 동작:

1. 저장값을 읽는다.
2. 스키마 버전을 확인한다.
3. 유효하지 않은 값은 기본값으로 보정한다.
4. 마지막 업데이트 시각 기준으로 오프라인 decay를 적용한다.
5. 보정된 상태를 다시 저장한다.

### 7.8 오프라인 decay

사용자가 앱을 닫은 시간도 펫 상태에 반영한다.

원칙:

- decay는 순수 함수로 계산한다.
- 최대 반영 시간은 제한한다.
- 너무 긴 미접속으로 모든 스탯이 한 번에 0이 되는 경험은 피한다.

MVP 기준:

- decay 기준 단위: 시간
- 최대 반영 시간: 24시간
- `fullness`, `mood`, `energy`가 시간에 따라 감소

### 7.9 스킨/테마 선택 UI

스킨/테마는 MVP에서 실제 확장 가능한 구조만 만든다.

포함:

- 기본 테마
- 포커스 테마
- 나이트 테마
- 파스텔 테마

테마는 본문 UI 색상과 favicon 색상에 영향을 줄 수 있다.

### 7.10 수익화 UI 목업

실제 광고 SDK와 결제는 제외하지만, 향후 수익화 검증을 위한 UI를 노출한다.

광고 보상형 목업:

- `Watch ad to restore energy`
- 클릭 시 실제 광고 없이 목업 모달 또는 토스트 표시
- 실제 스탯 보상 적용 여부는 MVP 구현 시 선택 가능하나, 광고 SDK는 붙이지 않는다.

프리미엄 목업:

- 추가 스킨 잠금 표시
- `Premium` 라벨
- 결제 플로우 없이 비활성 CTA 표시

### 7.11 이모지 복사 패널

사용자가 탭 제목이나 메모에 사용할 수 있는 이모지를 복사할 수 있게 한다.

예시:

- `🐱`
- `🐶`
- `💤`
- `🍗`
- `✨`
- `📊`
- `🗂`

복사 성공 시 짧은 피드백을 보여준다.

## 8. 핵심 사용자 시나리오

### 8.1 첫 사용

1. 사용자가 서비스에 접속한다.
2. 고양이/강아지 선택 화면을 본다.
3. 펫을 선택한다.
4. 기본 위장 타이틀과 테마가 적용된다.
5. 탭 제목과 favicon이 현재 상태를 반영한다.

### 8.2 일상 사용

1. 사용자가 탭을 열어둔다.
2. 시간이 지나면서 스탯이 감소한다.
3. 탭 title 또는 favicon이 변한다.
4. 사용자가 본문으로 돌아와 액션을 수행한다.
5. 스탯과 상태가 갱신되고 탭 반응도 즉시 바뀐다.

### 8.3 재방문

1. 사용자가 브라우저를 닫았다가 다시 접속한다.
2. 앱은 `localStorage`에서 이전 상태를 복원한다.
3. 닫혀 있던 시간만큼 decay를 반영한다.
4. 계산된 상태가 title/favicon에 적용된다.

## 9. 성공 기준

MVP 성공 기준:

- 사용자가 최초 진입 후 30초 이내에 펫을 선택하고 탭 변화를 확인할 수 있다.
- 액션 실행 후 title과 favicon이 즉시 변경된다.
- 새로고침 후에도 펫 상태가 복원된다.
- 하루 뒤 재방문 시 오프라인 decay가 반영된다.
- 앱의 핵심 경험이 본문 UI가 아니라 탭 반응이라는 점이 명확하다.

## 10. QA 기준

- SSR 단계에서 `document`, `window`, `localStorage` 접근 오류가 없어야 한다.
- `localStorage` 값이 깨져 있어도 앱이 기본 상태로 복구되어야 한다.
- 모든 스탯은 항상 0-100 범위여야 한다.
- 상태 계산은 저장된 상태값이 아니라 현재 스탯에서 도출되어야 한다.
- favicon link가 중복 생성되지 않아야 한다.
- title 프리셋 변경 시 현재 상태 시그널이 즉시 반영되어야 한다.
- 모바일/데스크톱 화면에서 주요 액션 버튼이 잘리지 않아야 한다.
